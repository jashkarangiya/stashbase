# Product Direction

StashBase's long-term product shape:

> Obsidian-like local file workspace + VS Code-like Agent Panel + Cursor-like document indexing.

## Obsidian-like Local File Workspace

- The main UI is centered on the user's own local folders.
- Preview and editing support reading, organizing, and maintaining local files.
- Markdown editing should feel closer to knowledge-base writing than code editing.
- StashBase should not introduce a heavy block, database, or page model.

## VS Code-like Agent Panel

- The agent works alongside the current folder and current file.
- The Agent Panel is a side panel inside the workspace, not a separate AI workspace.
- Context, permissions, and file operations should be clearly tied to the opened local folder.
- The built-in Agent Panel should not become a closed agent product.

## Cursor-like Document Indexing

- Opened folders should be converted and indexed into agent-usable context.
- Search should not depend only on filenames, paths, or user memory; semantic retrieval should be available when configured.
- Indexing serves user search and agent retrieval.
- The product surface should not become a complex search administration console.
